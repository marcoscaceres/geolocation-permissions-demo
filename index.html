Approximate Geolocation: Single vs. Two-Permission Model

A single geolocation permission with an accuracyMode hint satisfies every developer use case that a two-permission model can, without leaking the user's accuracy preference to third-party scripts.

Weather App

A weather app only needs city-level location. It declares what accuracy it needs and handles the result. Under both models, this works identically.

const status = await navigator.permissions.query({
  name: "geolocation"
});
navigator.geolocation.getCurrentPosition(
  pos => showWeather(pos.coords),
  err => showManualEntry(),
  {accuracyMode: "approximate"}
);

Upgrade Flow

A ride-sharing app starts with approximate location to show nearby drivers, then upgrades to precise when the user taps "Set pickup location." The user initiated the action; they expect interaction. The developer doesn't need to predict whether a prompt will appear — they call the API and handle the result. This is the same pattern as getUserMedia().

navigator.geolocation.getCurrentPosition(
  pos => showNearbyDrivers(pos),
  err => showManualAddress(),
  {accuracyMode: "approximate"}
);

navigator.geolocation.getCurrentPosition(
  pos => setPrecisePickup(pos),
  err => handleError(err),
  {accuracyMode: "precise"}

The enableHighAccuracy Precedent

Today, enableHighAccuracy is a PositionOptions hint with no separate queryable permission and no result attribute reporting what the UA did. query({name: "geolocation"}) returning "granted" has never guaranteed high-accuracy results. The developer checks coords.accuracy and adapts. accuracyMode is architecturally the same kind of thing.

const status = await navigator.permissions.query({
  name: "geolocation"
});
navigator.geolocation.getCurrentPosition(success, error, {
  enableHighAccuracy: true
});

The Privacy Cost

Under the two-permission model, any third-party script can silently learn the user's accuracy preference on page load without a user gesture. This compounds the concern raised in w3c/permissions#52 (open since 2015). The single-permission model doesn't leak this choice.

const approx = await navigator.permissions.query({
  name: "geolocation-approximate"
});
const precise = await navigator.permissions.query({
  name: "geolocation"
});
if (approx.state === "granted" && precise.state === "prompt") {
  trackUserPrivacyPreference("approximate-only");
}
